# Вариант 13 — Микро-шоп «Добавь в корзину» — Роли и действия

Краткое описание

MVP: Product, CartItem, Order, OrderItem, User, Address, Coupon, Payment. UI отображает каталог товаров, корзину, заказы; API: /products, /cart, /orders, /coupons, /payments. Приёмка (MVP): фильтры, купоны, CRUD товаров, оформление заказа.

Роли и основные действия

- Пользователь (USER)
  - Регистрация и вход в систему
  - Просмотр каталога товаров с фильтрами и поиском
  - Просмотр деталей товара
  - Добавление/удаление товаров в корзину
  - Изменение количества товаров в корзине
  - Применение купона на скидку
  - Оформление заказа с указанием адреса доставки
  - Просмотр истории своих заказов
  - Просмотр деталей заказа
  - Управление адресами доставки (добавление, редактирование, удаление)

- Администратор (ADMIN)
  - Все действия пользователя
  - CRUD товаров (создание, редактирование, удаление)
  - Управление категориями товаров
  - Просмотр всех заказов
  - Изменение статуса заказа
  - Управление купонами (создание, редактирование, активация/деактивация)
  - Просмотр списка пользователей
  - Блокировка/разблокировка пользователей
  - Просмотр статистики (продажи, популярные товары)

Примеры сценариев

- Пользователь просматривает каталог, применяет фильтры по категории и цене, добавляет товары в корзину, применяет купон, оформляет заказ с адресом доставки → заказ создан со статусом "pending".
- Администратор создаёт новый товар, устанавливает цену и количество на складе → товар появляется в каталоге для пользователей.
- Пользователь добавляет товар в корзину, изменяет количество, применяет купон "SUMMER10" → корзина обновляется с учётом скидки.
